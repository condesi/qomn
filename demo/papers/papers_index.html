Key Claims Summary

Proof 1 — Jitter Determinism (SCHED_FIFO, 10,000 ticks)
σ = 3,334 ns · p99 = 18,154 ns · 254× lower σ than untuned Linux baseline (SCHED_OTHER, ~850,000 ns σ).
Note: reference baseline is intentionally conservative (no CPU pinning, no RT tuning) — characterizes the problem class.
Proof 2 — SIMD Saturation (AVX2 + FMA)
154,439,021 scenarios/s · 0.0450 scenarios/clock-cycle · 16.9% of AVX2 theoretical max.
Utilization is memory-bandwidth limited (40KB SoA working set), not compute limited. Theoretical max assumes pure compute with no memory traffic.
Proof 3 — Adversarial Resilience (1.28M poison inputs)
0 panics · 0 undefined behavior · 61.2M/s throughput under attack · valid_frac drops 100%→50% as designed.
NaN×0=NaN per IEEE 754-2008 §6.2 is expected behavior. The critical guarantee is zero panics and no UB, not zero NaN outputs.
Proof 4 — Throughput Ratio vs. LLM Sequential Inference
Paper figure: 1,532,567,050× (conservative, based on 12s / 7.83ns/tick).
Measured: 1,853,268,252× (dedicated CPU benchmark).
170 Pareto-optimal solutions found in 1.84ms.
This ratio quantifies throughput for configuration space enumeration only. It is not a claim about reasoning quality.
